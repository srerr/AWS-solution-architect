# AWS Config:
-------------
    - AWS Config is a service that enables you to assess, audit, and evaluate the configurations of your AWS resources. With AWS Config, you can continuously monitor and record your AWS resource configurations and automate the evaluation of recorded configurations against desired configurations.
    - AWS Config provides a detailed view of the configuration of your AWS resources at any point in time. This includes information about the relationships between resources and how they are configured. This information can be used for security analysis, compliance auditing, change management, and operational troubleshooting.
    - AWS Config is enabled on an AWS account by default when you create the account. It delivers configuration snapshots and configuration history to an Amazon S3 bucket that you specify. You can also configure AWS Config to send configuration change notifications to Amazon SNS topics or to AWS Lambda functions for real-time processing.
    - AWS Config supports multi-region recording, which allows you to record configurations from all regions in a single account. This simplifies the management of your recordings and ensures that you have a comprehensive view of your resource configurations across all regions.
    - AWS Config also supports organization recording, which allows you to record configurations for all accounts in your AWS Organization. This provides a centralized view of resource configurations across your organization and simplifies compliance and auditing.
    - AWS Config integrates with other AWS services such as AWS Identity and Access Management (IAM), Amazon CloudWatch, and AWS Lambda to provide a comprehensive solution for monitoring and analyzing resource configurations. 
    - For example, you can use CloudWatch to create alarms based on specific configuration changes or patterns in your AWS Config logs, and you can use Lambda to automate responses to certain events captured in your AWS Config logs.
    - AWS Config provides a set of pre-built rules that you can use to evaluate the configurations of your AWS resources against best practices and compliance standards. You can also create custom rules using AWS Lambda functions to evaluate specific configuration changes or patterns in your AWS Config logs.
    - AWS Config also provides a feature called "Remediation" that allows you to automatically remediate non-compliant resources based on the rules you have defined. This can help you maintain compliance and ensure that your resources are configured according to your desired state.  
    - AWS Config also provides a feature called "Conformance Packs" that allows you to create and manage a collection of AWS Config rules and remediation actions as a single entity. Conformance Packs can be used to enforce compliance with specific regulatory standards or industry best practices across your AWS environment.    
    - AWS Config also provides a feature called "Aggregators" that allows you to aggregate configuration data from multiple accounts and regions into a single account. This can help you gain a centralized view of your resource configurations across your entire organization and simplify compliance and auditing. 
    - AWS Config also provides a feature called "Resource Inventory" that allows you to view and search the inventory of your AWS resources and their configurations. This can help you gain insights into your resource configurations and identify any potential issues or misconfigurations. 
    - AWS Config also provides a feature called "Configuration Recorder" that allows you to record the configuration changes of your AWS resources over time. This can help you track the history of your resource configurations and identify any changes that may have caused issues or security vulnerabilities. 
    - AWS Config also provides a feature called "Configuration Aggregator" that allows you to aggregate configuration data from multiple accounts and regions into a single account. This can help you gain a centralized view of your resource configurations across your entire organization and simplify compliance and auditing.    
    - AWS Config also provides a feature called "Configuration Snapshot" that allows you to capture the current configuration of your AWS resources at a specific point in time. This can help you create a baseline of your resource configurations and compare it against future configurations to identify any changes or deviations from your desired state.    
    - AWS Config also provides a feature called "Configuration History" that allows you to view the history of configuration changes for your AWS resources. This can help you track the changes made to your resources over time and identify any potential issues or misconfigurations.
    - AWS Config also provides a feature called "Configuration Compliance" that allows you to evaluate the compliance of your AWS resources against the rules you have defined. This can help you identify any non-compliant resources and take corrective actions to maintain compliance with your desired state.
